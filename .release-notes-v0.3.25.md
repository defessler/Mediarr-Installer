## Indexer browser + reinstall preserve

The long-running `feature/indexer-browser` branch finally lands on master. Plus a quality-of-life improvement for users re-running the wizard against an already-customised stack.

### Indexer browser

22 new builtin indexers, plus a unified searchable view that replaces the three legacy section blocks (Usenet / Private torrent / Bazarr providers) with one filterable catalog.

**New builtins**
- **Usenet:** NZB.su
- **Free public torrent:** Nyaa, SubsPlease, AniDex, Tokyo Toshokan, 1337x, TorrentGalaxy, The Pirate Bay, LimeTorrents, EZTV, TheRARBG, BitSearch, YTS
- **Private torrent:** TorrentLeech, HD-Torrents, BroadcasTheNet (BTN), MoreThanTV (MTV), PassThePopcorn (PTP), Redacted (RED), Orpheus Network

**`IndexerBrowser` UI**
- Text search (name + description)
- Four chip-filter rows:
  - **Content** — TV / Movies / Anime / K-drama / Asian (live) / Music / General
  - **Cost** — Free / Paid
  - **Signup** — No signup / Free signup / Invite-only / Application
  - **Kind** — Usenet / Torrent
- AND across axes, OR within each axis. "Reset filters" + empty-state hint.

**`CustomIndexerEditor`** for Newznab-compatible indexers not in the catalog. Per-entry form (name / URL / API key / categories / note), Add / Remove, plus an advanced "Edit JSON directly" textarea for power users. Persists to `CUSTOM_INDEXERS_JSON` in `.env`. Optional fields stay optional; malformed JSON shows a recoverable banner instead of nuking your entries.

`IndexerCard` sprouts a color-coded tag pill row so you can read each card's content/signup at a glance.

`setup-indexers.py` registers all new builtins through the existing add-newznab / add-private-indexer paths, AND parses `CUSTOM_INDEXERS_JSON` to register each user-defined entry as a Newznab indexer.

### Reinstall preserve mode

When you re-run the wizard against an already-customised stack, settings you tweaked in the Sonarr / Radarr / Plex / qBittorrent UI are now **preserved** instead of being stamped back to wizard defaults.

`setup-arr-config.py` drops `${INSTALL_DIR}/.wizard-stack-installed` after the first clean install. On subsequent runs the configure_* helpers switch into preserve mode:

| Setting | Behaviour on re-install |
|---|---|
| Auth (config/host) | Preserved when any customisation detected — different username, non-Forms method, etc. |
| Media-management TRaSH baselines | Per-key check vs. API defaults; only re-applies settings you never touched |
| Backup schedule | Hard-skipped |
| Plex QoL prefs (`GenerateBIFBehavior`, `ScheduledLibraryUpdatesEnabled`) | Hard-skipped (Manual port mapping still re-applied — that's external-reachability plumbing) |
| qBittorrent seeding limits (`max_ratio`, `max_seeding_time`) | Preserved; TMM-off flags still re-applied (load-bearing for arr Completed Download Handling) |

Stack-essential settings (bindAddress, indexer connections, root folders, Flaresolverr proxy, Plex Connect notification, qBit TMM-off) still re-apply every run — they're what holds the stack together.

**Escape hatch:** `rm <INSTALL_DIR>/.wizard-stack-installed` forces the next run to treat the stack as fresh-install. Every preserve-mode log line surfaces this hint.
